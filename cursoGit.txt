1. Clonar el repositorio sobre el que vamos a trabajar: git clone https://github.com/gmqdev/curso-git.git
2. Crear documento: notepad HolaMundo
3. Añadir al git: git add .
[Si no están configuradas las variables te pedirá que las añadas. Simplemente copiar y pegar lo que te dice cambiando el ejemplo por tu nombre de usuario y correo]
4. Hacer commit: git commit -a -m "Descripción de las modificaciones"
5. Subir a GitHub: git push origin master
[pedirá usuario y clave en una ventana emergente]

6. Invitar a otro usuario: Settings > Collaborators > Search by username... > Add Collaborator

CASO PRÁCTICO: El equipo está trabajando sobre el mismo fichero
1. Lo primero es hacer un pull: git pull
2. Uno de los compañeros crea un fichero: notepad autores.txt
3. Este mismo lo comparte:
git add .
git commit -a -m "Añadido fichero autores.txt"
git push origin master
4. Los demás compañeros se añaden al fichero autores
5. Ahora tengo que hacer un pull para actualizar los ficheros compartidos: git pull
6. Si intentamos editar los dos el mismo fichero ocurrirá que a alguno nos de un error tipo "error: failed to push some refs to ...". Entonces deberemos volver a un commit anterior:
